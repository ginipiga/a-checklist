# 📋 체크리스트 중심 프로젝트 관리 가이드

## 🎯 핵심 개념

이 시스템은 **체크리스트 중심**으로 설계되었습니다!

### 기본 구조
```
📄 프로젝트명
├─ 📋 섹션 1
│   ├─ ☑ 구체적인 작업 1 (10점)
│   ├─ ☑ 구체적인 작업 2 (8점)
│   └─ ☑ 구체적인 작업 3 (12점)
├─ 📋 섹션 2
│   ├─ ☑ 구체적인 작업 1 (15점)
│   └─ ☑ 구체적인 작업 2 (10점)
└─ ✅ 완료 조건
    └─ ☑ 최종 검증 (20점)
```

**왜 체크리스트 중심인가?**
- ✅ 진행률 추적이 명확
- ✅ 실행 가능한 작업 위주
- ✅ 복잡한 계층 구조 불필요
- ✅ 팀원 모두가 쉽게 이해

---

## 🚀 사용 방법

### 1. 기본 모드 (고정 템플릿)

```bash
실행.bat
```

파일을 드롭하면 다음 구조로 변환:
- 📋 문서 검토 및 분석 (체크리스트 3개)
- 📊 실행 계획 수립 (체크리스트 3개)
- 📄 문서 원본 내용
- ⚠️ 리스크 및 이슈 관리 (체크리스트 3개)
- ✅ 검증 및 완료 (체크리스트 4개)

**장점**: 빠르고 간단, 모든 문서에 일관된 구조
**단점**: 문서 내용에 맞춤화되지 않음

---

### 2. AI 스마트 분석 (Ollama - 권장!)

```bash
# 1. Ollama 설치 및 모델 다운로드
ollama pull llama3.2:latest

# 2. AI 모드로 실행
실행_AI_Ollama.bat
```

**AI가 자동으로 해주는 것:**

✨ **문서 유형 파악**
- 제안서 → 프로젝트 관리 구조
- 기술 문서 → 개발 및 배포 구조
- 보고서 → 분석 및 후속조치 구조

✨ **맞춤형 섹션 생성**
```
예: 신공항 건설 제안서

📄 인천국제공항 제3여객터미널 건설 제안서
├─ 📋 사업 타당성 검증
│   ├─ ☑ 수요 예측 모델 검증 및 승인 (15점)
│   ├─ ☑ 경제성 분석 보고서 작성 (12점)
│   └─ ☑ 이해관계자 의견 수렴 워크샵 개최 (10점)
├─ 🏗️ 기술 설계 및 검토
│   ├─ ☑ 터미널 배치 계획 수립 (18점)
│   ├─ ☑ 활주로 용량 분석 완료 (15점)
│   └─ ☑ 환경영향평가 초안 작성 (20점)
├─ 📅 인허가 및 법적 검토
│   ├─ ☑ 국토부 사업 승인 신청 (20점)
│   ├─ ☑ 환경부 환경영향평가 협의 (18점)
│   └─ ☑ 지자체 개발 허가 획득 (15점)
└─ ⚠️ 리스크 대응 전략
    ├─ ☑ 예산 초과 리스크 대응 계획 (15점)
    └─ ☑ 공정 지연 방지 방안 수립 (12점)
```

✨ **실행 가능한 체크리스트**
- "검토" ❌ → "환경영향평가 보고서 초안 작성 및 내부 검토" ✅
- "회의" ❌ → "이해관계자 워크샵 개최 및 의견 수렴" ✅
- "승인" ❌ → "국토부 사업 계획 승인 신청 및 획득" ✅

✨ **똑똑한 가중치**
- 단순 확인: 3-5점
- 일반 작업: 6-10점
- 중요 작업: 11-15점
- 핵심 작업: 16-20점

## 🔄 작업 흐름

### 1. 문서 드롭
```
PDF/Word/Excel 파일을 프로그램에 드래그 앤 드롭
```

### 2. AI 분석 (Ollama 사용 시)
```
🤖 AI가 문서를 분석하여 최적의 프로젝트 구조를 생성합니다...
✅ AI 분석 완료! 맞춤형 프로젝트 구조가 생성되었습니다.
```

### 3. 구조 확인 및 수정
- 생성된 섹션과 체크리스트 확인
- 필요시 "+ 블록 추가"로 체크리스트 추가
- 점수 조정

### 4. 진행률 추적
- 작업 완료 시 체크박스 클릭
- 자동으로 진행률 계산 및 표시
- 상위 토글에 자동 합산

### 5. 저장
- 💾 버튼으로 개별 토글 저장
- "모든 토글을 폴더에 저장"으로 전체 저장

## 🎓 추가 팁

### 체크리스트 세분화
하나의 작업이 너무 크면:
```
❌ "프로젝트 완료" (너무 큼)

✅ 다음으로 세분화:
- 테스트 완료 및 버그 수정 (12점)
- 문서 작성 및 검토 (8점)
- 최종 승인 획득 (10점)
- 배포 및 모니터링 (8점)
```

### 점수의 의미
- 점수 = 중요도 × 난이도
- 전체 프로젝트에서 차지하는 비중
- 100점 만점 기준으로 조정 가능

### 하위 토글 최소화
```
❌ 복잡한 구조:
프로젝트
└─ 개발
    └─ 백엔드
        └─ API
            └─ 인증
                └─ 체크리스트...

✅ 깔끔한 구조:
프로젝트
├─ 백엔드 API 개발
│   └─ 체크리스트...
├─ 프론트엔드 개발
│   └─ 체크리스트...
└─ 인증 시스템 구축
    └─ 체크리스트...
```
